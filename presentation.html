Slide 2 — Foundation

SCSS Architecture — The 7-1 Pattern

We structured our stylesheets following a professional modular pattern: one entry file that imports everything in logical order.

scss/
  abstracts/
    _variables.scss ← all tokens
    _mixins.scss ← utilities
  base/
    _reset.scss
    _fonts.scss
    _typography.scss
  layout/
    _app-layout.scss
    _sidebar.scss
    _header.scss
  components/
    _card.scss
    _nav-pill.scss
  pages/
    _home.scss, _search.scss…
  main.scss ← imports all

Why this structure?

  • Each file has one responsibility
  • Easy to find and edit any style
  • New pages = just add one partial
  • Compiles to a single CSS file

Sass Modules (@use)

// main.scss — modern @use syntax
@use 'abstracts/variables' as *;
@use 'abstracts/mixins' as *;
@use 'base/reset';
@use 'components/card';

Using @use (not @import) — the modern, non-deprecated Sass approach with proper scoping.

Slide 3 — Design Tokens

SCSS Variables — One Source of Truth

Every color, spacing value, size, and z-index is defined once in _variables.scss. Change it once — it updates everywhere.

/* Colors */
$color-bg:         #191414;
$color-accent:     #1DB954; ← Spotify green
$color-text:       #FFFFFF;
$color-text-muted: #B3B3B3;
$color-surface:    #282828;

/* Spacing — base 10px system */
$spacing-sm:   0.8rem;  // 8px
$spacing-base: 1.0rem;  // 10px
$spacing-md:   1.2rem;  // 12px
$spacing-xl:   2.4rem;  // 24px

/* Sizing from Figma */
$sidebar-width: 25.9rem; // 259px
$card-width:    16.0rem; // 160px

/* Z-index layers */
$z-header: 10;
$z-player: 20;
$z-modal:  30;

Why variables?

  • Consistent spacing across all pages
  • Update the accent color in one place
  • Figma measurements become named values
  • Z-index collisions are impossible
  • Font scale is systematic, not random
Slide 4 — Reusability

Sass Mixins — Write Once, Use Everywhere

Mixins are reusable Sass functions. Instead of repeating the same 3–5 lines across every file, we include a named mixin once.

// flex-center — used everywhere
@mixin flex-center {
  display:          flex;
  align-items:     center;
  justify-content: center;
}

// flex with parameters
@mixin flex($direction: row, $justify: flex-start,
           $align: center, $gap: 0) {
  display:         flex;
  flex-direction:  $direction;
  justify-content: $justify;
  align-items:     $align;
  gap:             $gap;
}

// truncate long text
@mixin truncate {
  overflow:      hidden;
  text-overflow: ellipsis;
  white-space:   nowrap;
}

// screen reader only (accessibility)
@mixin sr-only {
  position: absolute;
  width: 1px; height: 1px;
  clip: rect(0, 0, 0, 0);
}
// Usage — clean and readable
.nav-pill {
  @include flex-row($spacing-sm);
  width: $nav-pill-width;
}

.card__title {
  @include truncate;
  font-size: $font-size-label;
}

// grid-table mixin — liked songs page
@mixin grid-table($columns, $gap: 0, $align: center) {
  display:               grid;
  grid-template-columns: $columns;
  align-items:          $align;
  gap:                   $gap;
}

// called as:
.liked-song__row {
  @include grid-table(4.2rem 1fr 1fr 7rem 3rem 1fr);
}

8 Mixins in the project

  • flex-center — center anything
  • flex-row — row with gap
  • flex — full parametric flex
  • grid-table — grid layouts
  • truncate — ellipsis overflow
  • nav-column-item — icon+label stack
  • sr-only — accessibility
  • text-subtle — muted text style
Slide 5 — Dynamic Theming

CSS Custom Properties — Runtime Variables

SCSS variables are resolved at compile time. CSS Custom Properties (var()) are resolved at runtime — letting us change values per-element with just a class.

// _card.scss — accent bar uses CSS variable
.card__accent-bar {
  background: var(--accent-color, $color-accent);
}

// Each accent class sets the variable
$card-mix-accent-colors: (
  'yellow':     #FFFF00,
  'gold':       #FFD700,
  'cyan':       #00BFFF,
  'violet':     #8A2BE2,
  'pink':       #FF69B4,
  'orange':     #FF8C00,
  'lime':       #00FF7F,
  'orange-red': #FF4500,
);

@each $name, $color in $card-mix-accent-colors {
  .card--accent-#{$name} {
    --accent-color: #{$color};
  }
}
/* HTML — just add a class */
<article class="card card--accent-yellow">
<article class="card card--accent-violet">
<article class="card card--accent-cyan">

Why the hybrid approach?

  • SCSS vars → consistent global defaults
  • CSS vars → per-card color override
  • var(--name, fallback) = always safe
  • 8 accent colors from one @each loop
  • Add a new color = one line of Sass

Accent color preview

Each card picks its own color — zero extra CSS written.

Also used for frame color

$card-stack-frame-color:
  var(--card-stack-frame-color,
      #{$color-bg-app});

The stacked card frame border can be overridden per-context without touching SCSS.

Slide 6 — HTML & BEM

BEM Naming — Predictable, Scalable HTML

We used BEM (Block, Element, Modifier) so every class name tells you exactly what it does and where it belongs.

<!-- BEM anatomy -->
<!-- .block__element--modifier -->

<!-- Sidebar -->
<aside class="sidebar">
  <div  class="sidebar__header">
  <li   class="sidebar__item">
  <li   class="sidebar__item--arrow">
  <li   class="sidebar__item--accordion">
  <button class="sidebar__accordion-trigger">

<!-- Stacked card -->
<article class="card card--stacked card--accent-yellow">
  <div  class="card__thumb-wrap">
    <span class="card__mix-layer card__mix-layer--rear">
    <span class="card__mix-layer card__mix-layer--middle">
    <div  class="card__mix-front">
  <div  class="card__info">
    <h3  class="card__title">
    <p   class="card__desc">

BEM rules we followed

  • .block — standalone component
  • .block__element — child of block
  • .block--modifier — variant / state
  • Never use IDs for styling
  • Classes only — no tag selectors

Semantic HTML choices

  • <aside> for sidebar
  • <header> for top nav
  • <nav> for navigation
  • <article> for cards
  • <main> for page content
  • <button> for interactions

Accessibility built-in

  • aria-expanded="false" on accordion
  • aria-current="page" on active nav
  • aria-hidden="true" on decorative icons
  • @mixin sr-only for hidden labels
Slide 7 — Layout

Flexbox & Grid — The Right Tool for the Job

We chose flexbox or grid intentionally — not randomly. Each layout context uses whichever fits the problem best.

Flexbox — directional flows

/* App shell — sidebar beside content */
.app-layout {
  display:   flex;
  flex-direction: row;
  height:    100vh;
}
.app-layout__content {
  flex: 1; /* grows to fill remaining space */
  flex-direction: column;
}

/* Cards in a row — wrap on overflow */
.home__shelf-row {
  display: flex;
  gap:     $spacing-xl;
  flex-wrap: wrap;
}

/* Nav pill — icon + label side by side */
.nav-pill {
  @include flex-row($spacing-sm);
  width: $nav-pill-width;
}

CSS Grid — 2D alignment

/* Mobile nav — 5 equal columns */
.nav-mobile__inner {
  display:               grid;
  grid-template-columns: repeat(5, 1fr);
  align-items:          stretch;
}

/* Liked songs — table layout */
.liked-song__row {
  @include grid-table(
    4.2rem 1fr 1fr 7rem 3rem 1fr
  );
}

/* When to use Grid vs Flex:
   Flex  → 1D: a row or column of things
   Grid  → 2D: rows AND columns together  */

Key flex/grid decisions

  • flex: 1 → content fills available space
  • repeat(5, 1fr) → always-equal nav items
  • align-items: stretch → equal-height columns
  • gap spacing — never use margin hacks
Slide 8 — Advanced CSS

The Stacked Card Effect — Pure CSS

The Discover Weekly / Mix cards have a 3D stacked appearance. Three absolutely-positioned layers + transform + filter — no images, no JS.

/* Variables tune every layer precisely */
$stack-rear-rest-transform:
  translateY(-1.14rem) scale(0.83);
$stack-middle-rest-transform:
  translateY(-0.38rem) scale(0.92);
$stack-rear-rest-opacity:   0.56;

/* Images are desaturated at rest... */
$stack-layer-image-filter:
  saturate(0.78) brightness(0.46) contrast(1.02);

/* ...and restored on hover */
$stack-layer-image-hover-filter:
  saturate(1) brightness(1) contrast(1);

.card:is(:hover, :focus-visible) {
  .card__mix-layer-image {
    filter: $stack-layer-image-hover-filter;
    transition: filter 180ms cubic-bezier(0.22,1,0.36,1);
  }
}

How the layers work

  • 3 layers absolutely positioned in same space
  • Each layer has different translateY + scale
  • Rear: small, low, desaturated
  • Middle: mid-size, partially visible
  • Front: full size, bright, on top
  • Hover: layers shift + images brighten

CSS tricks used

  • isolation: isolate — prevents z-index leaks
  • box-shadow frame border per layer
  • cubic-bezier(0.22,1,0.36,1) — smooth spring
  • :is(:hover,:focus-visible) — modern selector
  • 20+ variables to tune the effect precisely

Discover Weekly variant

  • Uses ::before pseudo-elements
  • Overlay PNG applied via CSS background
  • Different opacity values per layer
Slide 9 — Responsive

Responsive Design — 4-Tier Breakpoint System

One codebase, every screen. The layout adapts from mobile to wide desktop using four precisely chosen breakpoints.

/* Breakpoints stored as variables */
$bp-mobile:  767px;  // ≤767px  — mobile
$bp-tablet:  768px;  // 768px+ — tablet
$bp-desktop: 992px;  // 992px+ — desktop
$bp-sidebar: 1245px; // 1245px+ — wide

/* Mobile: swap header nav for bottom nav */
@media (max-width: #{$bp-mobile}) {
  .app-header  { display: none; }
  .sidebar     { display: none; }
  .nav-mobile  { display: flex; }
}

/* Tablet: nav pills collapse to icon-only */
@media (min-width: $bp-tablet) and
       (max-width: 991px) {
  .nav-pill {
    width: $nav-item-size; // square, no label
  }
  .nav-pill__label { display: none; }
}

/* Responsive sizing with clamp() */
.discover__hero {
  width:  clamp(25rem, 35vw, 39rem);
  height: clamp(35rem, 60vw, 61.2rem);
}

What changes per breakpoint

  • Mobile: bottom nav, no sidebar, compact cards
  • Tablet: icon-only nav pills, no labels
  • Desktop: full header, sidebar visible
  • Wide desktop: sidebar always expanded

Modern CSS: clamp()

/* min, ideal, max — fluid in between */
clamp(25rem, 35vw, 39rem)
// Replaces 3 separate media queries!

Mobile-first decisions

  • Safe area inset for iPhone notch:
    padding-bottom: env(safe-area-inset-bottom)
  • Stacked card transforms tuned separately for mobile screen sizes
  • Feed modal hidden on mobile — different UX pattern
Slide 10 — CSS Magic

Notable CSS Techniques

Small details that add up to a polished result.

1:1 Aspect Ratio Trick

.card__thumb-wrap {
  position:   relative;
  padding-top: 100%;
}
.card__thumb {
  position:  absolute;
  inset:     0;
  object-fit: cover;
}

Always-square thumbnails regardless of image dimensions.

CSS-only icon swap

.icon--on  { display: none; }

.nav-pill--active {
  .icon--off { display: none; }
  .icon--on  { display: block; }
}

Active/inactive icon states — no JavaScript needed.

Accordion — max-height

.sidebar__dropdown {
  max-height: 0;
  overflow:   hidden;
  transition: max-height 0.3s ease;
}
.is-open .sidebar__dropdown {
  max-height: 20rem;
}

Smooth open/close animation without JavaScript layout hacks.

Glass-morphism modal

.feed-modal {
  background:
    rgba(61,59,50, 0.2);
  backdrop-filter:
    blur(100px);
  -webkit-backdrop-filter:
    blur(100px);
}

Frosted glass effect with Safari prefix for compatibility.

Sass color.adjust()

@use 'sass:color';

.filter-pill:hover {
  background:
    color.adjust(
      $color-surface,
      $lightness: 8%
    );
}

Programmatically lighten a color — no need to hardcode hex values.

Reduced motion

@media (prefers-reduced-motion: reduce) {
  .feed-modal {
    transition: none;
  }
}

Respects user's OS accessibility preference — no motion if they ask for it.
